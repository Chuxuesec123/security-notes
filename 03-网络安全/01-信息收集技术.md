# 信息收集技术 — 网络安全学习笔记

## 一、信息收集概述

### 1.1 什么是信息收集

信息收集（Information Gathering）是渗透测试和网络安全评估的第一步，也是最关键的一步。**"你掌握的信息越多，攻击面就越清晰，成功概率就越高。"** 信息收集的目标是获取目标系统的各类信息，为后续的漏洞分析和利用奠定基础。

### 1.2 信息收集的分类

| 分类 | 特点 | 是否与目标交互 | 风险 |
|------|------|---------------|------|
| **被动信息收集** | 通过公开渠道获取信息 | 否 | 极低，难以溯源 |
| **主动信息收集** | 直接与目标系统交互 | 是 | 较高，可能被记录 |

### 1.3 信息收集的维度

```
目标信息收集层次：
┌─────────────────────────────────────┐
│          组织与人员信息              │  ← 社会工程学层面
├─────────────────────────────────────┤
│          域名与子域名信息            │  ← 网络边界层面
├─────────────────────────────────────┤
│          IP地址与网段信息            │  ← 基础设施层面
├─────────────────────────────────────┤
│          端口与服务信息              │  ← 暴露面层面
├─────────────────────────────────────┤
│          Web应用与中间件信息         │  ← 应用层面
├─────────────────────────────────────┤
│          操作系统与设备指纹          │  ← 系统层面
├─────────────────────────────────────┤
│           漏洞与弱点信息             │  ← 攻击入口层面
└─────────────────────────────────────┘
```

## 二、被动信息收集

### 2.1 DNS 信息收集

DNS（域名系统）是互联网的基础设施，其中包含了大量有价值的信息。

#### 2.1.1 WHOIS 查询

WHOIS 用于查询域名的注册信息，包括注册人、联系方式、注册商、注册日期、过期日期等。

**常用命令：**

```bash
# Linux
whois example.com

# 在线查询网站
https://whois.icann.org
https://www.whois.com
```

**关键信息提取：**
- 注册人姓名、邮箱、电话
- 注册商信息
- DNS 服务器地址
- 注册日期和过期日期

> **💡 技巧：** 通过 WHOIS 信息可以找到同一注册人的其他域名，扩大攻击面。

#### 2.1.2 DNS 记录枚举

DNS 记录类型：

| 记录类型 | 含义 | 用途 |
|---------|------|------|
| A | IPv4 地址记录 | 将域名解析到 IPv4 |
| AAAA | IPv6 地址记录 | 将域名解析到 IPv6 |
| CNAME | 别名记录 | 域名别名映射 |
| MX | 邮件交换记录 | 邮件服务器地址 |
| NS | 名称服务器记录 | DNS 服务器 |
| TXT | 文本记录 | 存放 SPF、DKIM 等验证信息 |
| SOA | 起始授权记录 | 区域主 DNS 服务器信息 |

**工具：**

```bash
# dig 命令（最强大的 DNS 查询工具）
dig example.com A          # 查询 A 记录
dig example.com MX         # 查询 MX 记录
dig example.com NS         # 查询 NS 记录
dig example.com ANY        # 查询所有记录（部分 DNS 服务器已禁用）
dig @8.8.8.8 example.com   # 指定 DNS 服务器查询

# nslookup（Windows 内置）
nslookup example.com
nslookup -type=mx example.com
nslookup -type=ns example.com

# host 命令
host example.com
host -a example.com
```

#### 2.1.3 DNS 区域传输

DNS 区域传输本应用于主从 DNS 服务器之间同步数据，但如果配置不当，任何人都可以请求区域传输，从而获取整个域的所有 DNS 记录。

```bash
# 使用 dig
dig @ns1.example.com example.com AXFR

# 使用 host
host -l example.com ns1.example.com
```

> **⚠️ 注意：** 大多数现代 DNS 服务器已禁止区域传输，但仍值得尝试。

#### 2.1.4 子域名枚举

子域名反映了目标系统的组织结构，往往存在测试环境、开发环境等易受攻击的入口。

**方法一：字典爆破**

```bash
# 使用 Sublist3r
sublist3r -d example.com

# 使用 subfinder
subfinder -d example.com

# 使用 dnsenum
dnsenum example.com

# 使用 ffuf（基于 DNS）
ffuf -w subdomains.txt -u http://example.com -H "Host: FUZZ.example.com" -fs 0
```

**方法二：搜索引擎**

```text
site:example.com -www
```

**方法三：证书透明度日志（Certificate Transparency）**

证书透明度是记录 SSL/TLS 证书的公开日志系统，任何人都可以查询到某个域名下签发的所有证书，从中提取子域名。

```bash
# 使用 crt.sh 网站
# https://crt.sh/?q=%25.example.com

# 使用 curl 查询
curl -s "https://crt.sh/?q=%25.example.com&output=json" | jq .

# 使用工具
certspotter -d example.com
```

**方法四：DNS 记录聚合查询**

```bash
# 使用 dnsrecon
dnsrecon -d example.com -t std  # 标准枚举
dnsrecon -d example.com -t brt  # 爆破枚举
dnsrecon -d example.com -D subdomains.txt -t brt
```

### 2.2 搜索引擎信息收集

#### 2.2.1 Google Hacking（Google Dork）

Google 高级搜索运算符是信息收集的利器。

| 运算符 | 语法示例 | 说明 |
|--------|---------|------|
| `site:` | `site:example.com` | 限定搜索域名 |
| `inurl:` | `inurl:admin` | URL 包含指定关键字 |
| `intitle:` | `intitle:"index of"` | 标题包含指定关键字 |
| `filetype:` | `filetype:pdf` | 搜索指定文件类型 |
| `intext:` | `intext:"password"` | 页面正文包含关键字 |
| `link:` | `link:example.com` | 搜索链接到目标页面的页面 |
| `cache:` | `cache:example.com` | 查看 Google 缓存 |
| `related:` | `related:example.com` | 查找相关网站 |

**经典 Google Dork 示例：**

```text
# 目录遍历漏洞
intitle:"index of" site:example.com

# 敏感文件
site:example.com filetype:sql
site:example.com filetype:env
site:example.com filetype:bak
site:example.com filetype:log
site:example.com filetype:xls "username" "password"

# 管理后台
site:example.com inurl:admin
site:example.com inurl:login
site:example.com intitle:"后台管理"

# 配置文件
site:example.com inurl:"phpinfo.php"
site:example.com inurl:"config.php"

# 错误信息泄露
site:example.com "Warning: mysql_connect()"
site:example.com "PHP Parse error"
site:example.com "Fatal error:"
```

## 三、主动信息收集

### 3.1 主机发现

#### 3.1.1 Ping 扫描

```bash
# 单个主机
ping -c 4 192.168.1.1

# Nmap Ping 扫描

nmap -sn -PS22,80,443 192.168.1.0/24

nmap -sn -PU 192.168.1.0/24        # UDP Ping
```

```bash
# Nmap ARP 扫描（同网段最快）

netdiscover -r 192.168.1.0/24
netdiscover -i eth0 -r 192.168.1.0/24
```

### 3.2 端口扫描

#### 3.2.1 TCP 扫描技术

| 扫描类型 | Nmap 参数 | 原理 | 特点 |
|---------|----------|------|------|
| TCP SYN 扫描 | `-sS` | 发送 SYN，收到 SYN/ACK 即开放 | 半连接，速度快，隐蔽 |
| TCP Connect 扫描 | `-sT` | 完整三次握手 | 速度慢，易被记录 |
| TCP ACK 扫描 | `-sA` | 发送 ACK 包 | 用于探测防火墙规则 |
| TCP FIN 扫描 | `-sF` | 发送 FIN 包 | 用于穿透非 Windows 防火墙 |
| TCP NULL 扫描 | `-sN` | 空标志位 | 用于穿透非 Windows 防火墙 |
| TCP Xmas 扫描 | `-sX` | FIN+PSH+URG 标志 | 用于穿透非 Windows 防火墙 |
| TCP Window 扫描 | `-sW` | 利用窗口大小判断端口状态 | 特殊场景使用 |
| TCP Maimon 扫描 | `-sM` | FIN/ACK 标志 | 特殊场景使用 |

#### 3.2.2 UDP 扫描

```bash
# UDP 扫描（速度慢，但能发现 UDP 服务）
nmap -sU 192.168.1.100
nmap -sU -p 53,67,68,123,161 192.168.1.100
```

#### 3.2.3 Nmap 实战

```bash
# 基本扫描
nmap 192.168.1.100                          # 默认扫描 1000 个常用端口
nmap -p- 192.168.1.100                      # 扫描全部 65535 个端口
nmap -p 1-10000 192.168.1.100               # 扫描指定范围端口
nmap -p 22,80,443,3306,8080 192.168.1.100   # 扫描指定端口列表

# 服务版本识别
nmap -sV 192.168.1.100                      # 识别服务版本
nmap -sV --version-intensity 9 192.168.1.100  # 设置版本检测强度（0-9）

# 操作系统识别
nmap -O 192.168.1.100                       # 操作系统检测
nmap -O --osscan-guess 192.168.1.100        # 激进猜测模式

# 综合扫描
nmap -A 192.168.1.100                       # 全面扫描（OS + 版本 + 脚本 + traceroute）
nmap -sS -sV -O -T4 192.168.1.0/24         # 综合扫描网段

# 输出控制
nmap -oN result.txt 192.168.1.100           # 正常格式输出
nmap -oX result.xml 192.168.1.100           # XML 格式输出
nmap -oG result.gnmap 192.168.1.100         # Grep 可解析格式
nmap -oA result 192.168.1.100               # 全部格式输出

# 时序和性能
nmap -T0 192.168.1.100                      # 极慢（IDS 规避）
nmap -T4 192.168.1.100                      # 快速（常规使用）
nmap -T5 192.168.1.100                      # 极快（可能漏包）

# NSE 脚本扫描
nmap --script default 192.168.1.100         # 默认脚本
nmap --script vuln 192.168.1.100            # 漏洞检测脚本
nmap --script http-headers 192.168.1.100    # HTTP 头信息
nmap --script banner 192.168.1.100          # 横幅抓取
nmap --script dns-zone-transfer example.com  # DNS 区域传输
```

#### 3.2.4 Masscan（超高速扫描）

Masscan 可以在几分钟内扫描整个互联网。

```bash
# 安装
# apt install masscan

# 基本用法
masscan 192.168.1.0/24 -p80,443,8080       # 扫描指定端口
masscan 192.168.1.0/24 -p0-65535 --rate=1000  # 全端口扫描
masscan 10.0.0.0/8 -p443 --rate=10000      # 大规模扫描

# 输出
masscan 192.168.1.0/24 -p80 -oJ result.json
masscan 192.168.1.0/24 -p80 -oL result.txt
```

**Nmap vs Masscan 对比：**

| 特性 | Nmap | Masscan |
|------|------|---------|
| 速度 | 较慢（默认） | 极快（异步传输） |
| 功能丰富度 | 非常丰富 | 仅扫描 |
| 服务识别 | 支持 | 不支持 |
| 脚本引擎 | 支持 | 不支持 |
| OS 识别 | 支持 | 不支持 |
| 大规模扫描 | 不适合 | 专为大规模设计 |

### 3.4 操作系统指纹识别

#### 3.4.1 主动 OS 指纹识别

利用 TCP/IP 协议栈实现的微小差异来识别操作系统。

```bash
# Nmap OS 识别
nmap -O 192.168.1.100
nmap -O --osscan-guess 192.168.1.100

# 指定端口进行 OS 识别（需要开放和关闭的端口）
nmap -O -p 22,80,443 192.168.1.100
```

**指纹识别的关键指标：**

- **TTL 值**：Windows 默认 128，Linux 默认 64，路由器 255
- **TCP 窗口大小**：不同 OS 差异明显
- **DF 位**：是否设置不分片标志
- **初始序列号（ISN）**：不同 OS 生成模式不同

```bash
# 使用 ping 查看 TTL
ping 192.168.1.100
# TTL=64 → Linux/Mac
# TTL=128 → Windows
# TTL=255 → Cisco/网络设备
```

#### 3.4.2 被动 OS 指纹识别

```bash
# p0f（被动指纹识别）
p0f -i eth0
p0f -i eth0 -p

# 分析 pcap 文件
p0f -r capture.pcap
```

### 4.3 WAF 识别

#### 4.3.1 什么是 WAF

WAF（Web Application Firewall）是 Web 应用防火墙，用于检测和拦截恶意请求。在渗透测试前，识别 WAF 类型非常重要。

#### 4.3.2 WAF 识别方法

```bash
# Wafw00f
wafw00f http://example.com

# Nmap WAF 脚本
nmap --script http-waf-detect http://example.com
nmap --script http-waf-fingerprint http://example.com
```

**常见 WAF 特征：**

| WAF 类型 | 特征响应头/页面 |
|---------|---------------|

| F5 BIG-IP | `X-Cnection: close` |

| 阿里云 WAF | `Ali-Security-Token` |
| 腾讯云 WAF | `X-WAF` |

### 4.4 真实 IP 寻找

当目标使用了 CDN/WAF 时，需要找到源站的真实 IP。

#### 4.4.1 绕过 CDN 的方法

```text
1. 子域名枚举 — 有些子域名可能没有使用 CDN
2. 邮件头分析 — 查看邮件原文中的 Received 字段
3. DNS 历史记录 — 使用 SecurityTrails 等查看历史 DNS 记录
4. SSL 证书信息 — 通过 crt.sh 查找证书关联的 IP
5. 搜索引擎缓存 — Google 缓存的页面可能包含真实 IP
6. 第三方服务 — Shodan、FOFA 搜索域名关联的 IP
7. 社交媒体 — 内部人员泄露的真实 IP
```

#### 4.4.2 工具

```bash
# 使用 dig 查找 A 记录
dig example.com A

# 查找邮件服务器
dig example.com MX
dig example.com TXT

# 使用 onetrulyip（自动化绕过 CDN 查询）
https://github.com/SpiderLabs/OneTrueIp
```

## 六、信息收集方法论与流程

### 6.1 标准信息收集流程

```
第一阶段：信息收集启动
    ├── 明确目标范围（域名、IP 段）
    ├── 获取授权（书面）
    └── 了解目标行业和业务

第二阶段：被动信息收集
    ├── WHOIS / DNS 查询
    ├── 子域名枚举（证书透明度、搜索引擎）
    ├── 网络空间搜索引擎（Shodan / FOFA）
    ├── 社交媒体信息爬取
    └── 历史存档查询

第三阶段：主动信息收集
    ├── 主机发现（Ping / ARP）
    ├── 端口扫描（Nmap / Masscan）
    ├── 服务版本识别
    ├── 操作系统识别
    └── 漏洞扫描

第四阶段：Web 应用深度收集
    ├── 目录扫描
    ├── CMS / WAF / 中间件识别
    ├── JS 文件分析
    ├── 表单参数分析
    └── 认证机制分析

第五阶段：信息整理与分析
    ├── 绘制网络拓扑图
    ├── 标记已发现的漏洞入口
    ├── 整理用户密码字典
    └── 生成信息收集报告
```

### 6.2 信息收集清单

**基础信息：**
- [ ] 目标域名及注册信息
- [ ] 目标 IP 地址及网段
- [ ] DNS 服务器及 MX 记录

**网络信息：**

- [ ] 服务版本信息
- [ ] 操作系统类型及版本
- [ ] 网络拓扑结构

**Web 信息：**
- [ ] Web 服务器软件及版本
- [ ] CMS 类型及版本
- [ ] 编程语言及框架
- [ ] 敏感目录和文件
- [ ] WAF 类型及绕过方法

## 七、信息收集的合法性

### 7.1 法律与道德边界

```text
⚠️ 重要提示：

1. 未经授权的信息收集可能违反当地法律
2. 在中国大陆，《网络安全法》《数据安全法》《个人信息保护法》
   对信息收集行为有严格规定
3. 渗透测试必须在获得书面授权后进行
4. 公开信息 ≠ 可以随意使用
5. 发现漏洞应及时报告，而非利用
```

### 7.2 授权测试注意事项

- **获取书面授权**：明确测试范围、时间、方法
- **遵守测试范围**：不越界测试非授权目标
- **保护数据**：不下载、传播敏感数据
- **及时报告**：发现问题第一时间汇报
- **清理痕迹**：测试完成后清理所有工具和数据
