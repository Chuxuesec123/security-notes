
**网络是什么**

让两台电脑可以交换数据

例如访问百度，实际是如下操作：

```text
浏览器
    ↓
DNS解析域名
    ↓
获得IP地址
    ↓
TCP建立连接
    ↓
HTTPS加密
    ↓
发送HTTP请求
    ↓
服务器响应
    ↓
浏览器渲染页面
```

**ip地址**

类似门牌号，用来定位资源

例如IP地址：192.168.1.1，表示的是网络中的一台主机

其中127.0.0.1表示本机自己，也可以写成localhost

**私有IP地址**

经常见到：

```text
10.x.x.x

172.16.x.x ~ 172.31.x.x

192.168.x.x
```

这些都是内网地址

例如192.168.1.10

这些地址只能在局域网内使用

**端口**

一台主机可以运行很多服务

例如IP地址是：192.168.1.1的主机，上面运行了如下服务：

```text
80   web
443  https
3306 mysql
22   ssh
```

真正去访问这个主机的服务应该是：ip+port

```text
192.168.1.1:3306
```

**协议**

协议就是大家约定好的交流规则

比如http，网页通信：

```text
浏览器 ←→ 网站
```

比如https，http + ssl：

```text
浏览器 ←→ 网站
```

但是和http的区别是，中间人看不到内容。

dns协议负责把域名解析成IP地址：

```text
www.baidu.com 
↓
220.181.132.176
```

ssh协议，用来远程登陆服务器，端口是22

**osi七层模型**

面试很爱问，但是初学者不需要死记硬背

```text
应用层
表示层
会话层
传输层
网络层
数据链路层
物理层
```


应用层协议有：http、https、dns、ftp
传输层协议有：tcp、udp
网络层协议有：ip

**tcp三次握手**

建立连接前：

```text
客户端：我要连接你

服务器：收到

客户端：确认收到
```

过程：

```text
SYN
↓
SYN+ACK
↓
ACK
```

这就是三次握手

**TCP和UDP**

*重点*

TCP特点：可靠、有确认、不丢数据
例如：

```text
http
https
ssh
mysql
```

UDP特点：快、不确认、可能丢包
例如：

```text
视频
语音
游戏
DNS
```

**HTTP**

web安全密切接触

一个请求长这样：

```text
GET /index.php?id=1 HTTP/1.1
Host: test.com
User-Agent: Chrome
Cookie: admin=1
```

含义：

```text
GET        请求方法

/index.php 访问路径

Host       域名

Cookie     身份信息
```

服务器响应：

```text
HTTP/1.1 200 OK

<html>
Hello
</html>
```

**GET和POST**

很常见的两种请求方法

GET：参数在URL上

```text
?id=1

http://test.com/index.php?id=1
```

POST：参数在请求体中

```text
POST /login.php

username=admin
password=123456
```

**Cookie和Session**

这个和身份认证相关

登录网站：

```text
账号密码
↓
服务器验证
↓
给你Cookie
```

后面再次访问这个网站：

```text
Cookie
↓
证明你已经登录
```

例如：

```text
Cookie:
PHPSESSID=abc123
```

很多越权、会话劫持都和这个有关系

**DNS**

例如：www.baidu.com

浏览器其实不知道目标在哪里，因为前面说了，IP地址才是定位目标的工具

所以浏览器会问DNS服务器，谁是www.baidu.com

然后DNS就会返回这个服务器的IP地址
